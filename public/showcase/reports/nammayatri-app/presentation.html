Root Cause Analysis

Six Deep Dives into the Codebase

Each domain was analyzed through Haskell backend, React Native frontend, GitHub issues/PRs, and industry benchmarks.

Root Cause Analysis — Key Findings

Driver Cancellation

  • Structural, not behavioral. Ola found only 50% of cancellations are "driver's fault."
  • Race condition in cancellation lock (30s TTL)
  • Double-counting bug on rider cancel path
  • Same-driver reallocation bug (#1379, #1614)
  • Dual blocking systems (DriverScore + CancellationRate) create confusion
  • No passive cancellation detection (driver stationary)
  • Hardcoded AUTO_CATEGORY in penalty fee calc

Fare & Pricing

  • Congestion/surge charge not shown as line item
  • Fare range display confusing — always lands at upper end
  • Normal vs Priority Auto distinction unclear
  • Waiting charge triggered via GPS — no proximity enforcement
  • Toll charges not shown before booking
  • Rate card inaccuracy across cities (#7254, #7624)
  • Distance estimation diverges from actual route

Wait Time & Allocation

  • Initial search radius only 500m — too narrow
  • Radius step size 50m — too slow expansion
  • Driver batch size = 3 — too few per batch
  • 12s batch process time — too slow
  • Empty batches still wait full duration
  • PR #13979 (10-20x allocator speed) not merged
  • No peak-hour config overrides

Payment & Refund

  • No payment status check before ride cancellation
  • Double-charge possible — no idempotency guard
  • PENDING/timeout treated as failed
  • Auto-refund cron not implemented (#2708)
  • PaymentOrder not cancelled on cash switch
  • HyperSDK initialized per-ride, not app launch
  • Infinite loading states (#3762, #3769)

Safety & Driver Behavior

  • SOS buried in menus — not 1-tap accessible
  • Top 5 SOS bugs unfixed (#5424, #5297, #5311, #5483, #5432)
  • Route deviation alerts disabled during daytime
  • Night deviation threshold too high (3 waypoints)
  • No phone number masking (direct driver-rider calls)
  • No driver verification badge shown to rider
  • No consequence pipeline (warning→training→suspend)

Monitoring & Observability

  • Critical gap: No ride allocation Prometheus metrics
  • Critical gap: No driver cancellation Prometheus metrics
  • No GPS accuracy/drift monitoring
  • No alert rules as code (manual config)
  • No SLO definitions per user journey
  • Missing runbooks for top incident types
  • No mobile/client-side observability
