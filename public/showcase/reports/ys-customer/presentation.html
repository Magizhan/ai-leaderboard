Technical Root Causes

Payment, QR, GPS, and infrastructure failures

Payment — "Money Deducted, No Ticket"

#1 driver of 1-star reviews. Multiple open PRs attempt to fix architectural gaps.

Hardcoded Risk Values

  • Booking poll interval: 2000ms (rate limit risk)
  • Payment polling timeout: 30s
  • Quote validity buffer: 5 seconds
  • PAYMENT_PENDING timeout: NONE

Architectural gap:

If payment succeeds but ticket generation fails, there is no compensating transaction. Every case requires manual customer support — which is itself non-functional.

QR Ticketing — Failure Taxonomy

WBTC uses 3,056 EbixCash Android ETMs. Downtime is a recognized problem.

A. QR Generation Failures

  • Money deducted but no QR generated
  • App crashes before QR display
  • QR fails to load underground (no signal)
  • OTP activation timeout

B. QR Scanning Failures

  • ETM scanner cannot read QR (dirty lens, low brightness)
  • AFC gate hardware incompatibility
  • Static QR suspended by CMRL (Chennai)

C. Validation Failures

  • 30-minute ticket expiry with no refund path
  • Backend validation timeout
  • Server-side outage (common across all Beckn platforms)

D. Human/Process Failures

  • Conductor unfamiliarity with ETMs
    BMTC experience: #1 failure mode
  • Conductor refuses digital tickets
  • Peak-hour workload prevents scanning

Scale-up risk: Currently only 12-16 airport routes. These failures will explode as coverage expands city-wide.

GPS & Live Tracking — Two-Tier Problem

Government Buses (60 buses, 16 routes)

  • Dedicated Vehicle Tracking Units (VTU) hardware
  • Reliable but limited coverage
  • Activation requires conductor QR scan (not automatic)
  • AIS-140 VLTD (regulatory) and WIMB (consumer) on separate infra — no data sharing

Private Buses

  • Rely on driver mobile phone for tracking
  • Battery drain: app kills phone after ~2 trips
  • Drivers switch off phones → buses go dark
  • High data consumption (expensive for drivers)

Fundamental Data Gaps

  • No official GTFS data for Kolkata
  • Transit data fragmented across 5+ operators (WBTC, CTC, SBSTC, private)
  • "Where Is My Bus" covers only 60 of thousands of buses
  • No crowding or occupancy data

Ride-Hailing Location Issues

  • Premature "arrived" detection → inflated fare via waiting charges
  • Ghost bookings: driver shown nearby but never moves
  • Geofence radius too large
  • Geocoding errors (e.g., Chitpur station pointing wrong)

App Stability & Performance

White-Screen Bug (Dec 2025)

Post-update white screen persisted 2+ weeks, completely blocking all bookings. Unknown user impact. No crash reporting or alerting caught it — discovered via user complaints.

Missing Infrastructure

  • No Firebase Crashlytics or equivalent
  • No staged rollout or rollback capability
  • No crash reporting or alerting pipeline
  • Issues discovered via user complaints only

Driver App Stale

14-month gap in driver app updates (last: Jan 2025, v1.4.28). Driver-side bugs directly cause passenger-side symptoms like ghost bookings and location issues.

Open Issues in GitHub

  • 75+ issues found across 15+ repos
  • 40+ open, many dating back to Aug-Oct 2023
  • Namma Yatri branding leaking into Yatri Sathi
  • CUG subscription UI defects

The Three-Tier Problem

Issues span infrastructure, user experience, and human operations

Tier 1 — Infrastructure
  • Payment→ticket atomicity broken (no auto-refund, dead retry button)
  • Two-tier GPS tracking (govt VTU vs driver phones)
  • No GTFS data for Kolkata. No crash reporting pipeline
  • QR infrastructure unreliable (ETM downtime, 30-min expiry, no offline)
Tier 2 — User Experience
  • White-screen bug blocked bookings for 2+ weeks
  • Ghost bookings, premature arrival detection, inflated fares
  • Bus tracker advertised but covers only 60 of thousands of buses
  • 75+ open GitHub issues, some 2+ years old
Tier 3 — Operations
  • Driver cancellations (#1 complaint, 35%) and extortion unchecked
  • Customer support completely non-functional (email only: yatrisathi.support@wb.gov.in)
  • Insurance claims broken despite being advertised
  • Driver app not updated in 14 months
