  • P0 bundleTimeOut: 1000ms — premature SDK timeout
  • P0 Double process() call — duplicate charges

Root Causes: QR, GPS & iOS

QR Code Scanning

  • Payload 315-355 chars (industry: 60-80) → ~40% scan failure
  • Dark mode inverts QR colors → 0% scan success
  • Wrong-direction tickets from inverted direction bit
  • Bus OTP route mismatch after MTC renumbering

GPS / Location

  • Silent fallback to hardcoded Chennai Central coords
  • iOS stop proximity: 8m (industry min: 100m)
  • No age validation on cached locations
  • Bus tracking: 25m displacement filter → buses "disappear"
  • Chennai depot polygons not configured
  • Suburban booking requires physical station presence

iOS-Specific

  • FCM push notifications broken since Jan 2026
  • Batch interval 30s (Android: 10s)
  • Max batch age 300s (Android: 10s)
  • App size 178.9 MB (Android: ~91.5 MB)
  • WKWebView termination not handled

Accessibility (Legal Risk)

Screen reader non-functional. No low-floor bus filtering. Color contrast failures (2.88:1 vs 4.5:1 WCAG AA). RPwD Act 2016 non-compliant — penalties up to Rs 5,00,000.

Monitoring (Flying Blind)

71% of recommended alerts missing (14/49). Zero payment page load monitoring. Zero QR scan tracking. Zero GPS accuracy metrics. No formal SLOs defined.
